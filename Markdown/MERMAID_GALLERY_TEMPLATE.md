## üîÑ Flowchart Diagrams

<div style="background: white; border-left: 4px solid #06b6d4; padding: 24px; margin: 20px 0; border-radius: 8px; box-shadow: 0 2px 8px rgba(0,0,0,0.1);">

### Basic Flowchart
```mermaid
flowchart TD
  A[Start] --> B{Decision}
  B -->|Yes| C[Proceed]
  B -->|No| D[Stop]
  C --> E[Complete]
  D --> E
```

### Complex User Authentication Flow
```mermaid
flowchart TB
  Start([User Login Attempt]) --> CheckInput{Valid Input?}
  CheckInput -->|No| Error1[Show Validation Error]
  Error1 --> Start
  CheckInput -->|Yes| QueryDB[(Query Database)]
  QueryDB --> UserExists{User Found?}
  UserExists -->|No| Error2[User Not Found]
  Error2 --> Start
  UserExists -->|Yes| CheckPassword{Password Match?}
  CheckPassword -->|No| Attempts{Attempts < 3?}
  Attempts -->|Yes| Error3[Invalid Password]
  Error3 --> Start
  Attempts -->|No| LockAccount[Lock Account]
  LockAccount --> Notify[Send Email Notification]
  Notify --> End([Login Failed])
  CheckPassword -->|Yes| Check2FA{2FA Enabled?}
  Check2FA -->|No| CreateSession[Create Session]
  Check2FA -->|Yes| Verify2FA{2FA Code Valid?}
  Verify2FA -->|No| Error4[Invalid 2FA Code]
  Error4 --> Start
  Verify2FA -->|Yes| CreateSession
  CreateSession --> LogActivity[Log Login Event]
  LogActivity --> Success([Login Successful])
```

### Deployment Pipeline Flowchart
```mermaid
flowchart LR
  subgraph "Development"
    A[Code Commit] --> B[Run Unit Tests]
    B --> C{Tests Pass?}
    C -->|No| D[Notify Developer]
    D --> A
  end

  subgraph "CI/CD"
    C -->|Yes| E[Build Docker Image]
    E --> F[Push to Registry]
    F --> G[Deploy to Staging]
  end

  subgraph "Testing"
    G --> H[Integration Tests]
    H --> I{Tests Pass?}
    I -->|No| J[Rollback]
    J --> D
  end

  subgraph "Production"
    I -->|Yes| K{Manual Approval?}
    K -->|No| L[Wait for Approval]
    L --> K
    K -->|Yes| M[Deploy to Production]
    M --> N[Health Check]
    N --> O{Healthy?}
    O -->|No| P[Automatic Rollback]
    P --> D
    O -->|Yes| Q([Deployment Complete])
  end
```

### E-Commerce Checkout Sequence
```mermaid
sequenceDiagram
  autonumber
  actor Customer
  participant Frontend
  participant Backend
  participant Payment
  participant Inventory
  participant Email

  Customer->>Frontend: Click Checkout
  Frontend->>Backend: POST /api/checkout
  activate Backend
  Backend->>Inventory: Reserve Items
  activate Inventory
  Inventory-->>Backend: Reserved
  deactivate Inventory

  Backend->>Payment: Process Payment
  activate Payment
  Payment-->>Backend: Payment Success
  deactivate Payment

  Backend->>Inventory: Confirm Order
  activate Inventory
  Inventory-->>Backend: Order Confirmed
  deactivate Inventory

  Backend->>Email: Send Confirmation
  activate Email
  Email-->>Customer: Order Confirmation Email
  deactivate Email

  Backend-->>Frontend: Order Complete
  deactivate Backend
  Frontend-->>Customer: Show Success Page
```

### OAuth 2.0 Authorization Flow
```mermaid
sequenceDiagram
  participant User
  participant App as Client App
  participant Auth as Auth Server
  participant API as Resource API

  User->>App: Click "Login with OAuth"
  App->>Auth: Authorization Request
  Auth-->>User: Login Page
  User->>Auth: Credentials
  Auth->>Auth: Validate User
  Auth-->>User: Authorization Code
  User->>App: Redirect with Code

  rect rgb(168, 85, 247, 0.1)
    note right of App: Token Exchange
    App->>Auth: Exchange Code for Token
    Auth->>Auth: Verify Code
    Auth-->>App: Access Token + Refresh Token
  end

  loop API Requests
    App->>API: Request with Access Token
    API->>API: Validate Token
    API-->>App: Protected Resource
  end

  alt Token Expired
    App->>Auth: Refresh Token Request
    Auth-->>App: New Access Token
  else Token Invalid
    App->>User: Re-authenticate
  end
```

### Microservices Saga Pattern
```mermaid
sequenceDiagram
  participant Orchestrator
  participant Order
  participant Payment
  participant Inventory
  participant Shipping

  Orchestrator->>Order: Create Order
  activate Order
  Order-->>Orchestrator: Order Created
  deactivate Order

  Orchestrator->>Payment: Process Payment
  activate Payment

  alt Payment Success
    Payment-->>Orchestrator: Payment Confirmed
    deactivate Payment

    Orchestrator->>Inventory: Reserve Stock
    activate Inventory

    alt Stock Available
      Inventory-->>Orchestrator: Reserved
      deactivate Inventory

      Orchestrator->>Shipping: Create Shipment
      activate Shipping
      Shipping-->>Orchestrator: Shipment Scheduled
      deactivate Shipping

      note over Orchestrator: Transaction Complete
    else Stock Unavailable
      Inventory-->>Orchestrator: Out of Stock
      deactivate Inventory

      Orchestrator->>Payment: Refund
      activate Payment
      Payment-->>Orchestrator: Refunded
      deactivate Payment

      Orchestrator->>Order: Cancel Order
      activate Order
      Order-->>Orchestrator: Cancelled
      deactivate Order
    end

  else Payment Failed
    Payment-->>Orchestrator: Payment Failed
    deactivate Payment

    Orchestrator->>Order: Cancel Order
    activate Order
    Order-->>Orchestrator: Cancelled
    deactivate Order
  end
```

### Subgraphs for Logical Grouping
```mermaid
flowchart TB
  subgraph "User Interface"
    A[Login Page]
    B[Dashboard]
    C[Settings]
  end

  subgraph "Business Logic"
    D[Authentication Service]
    E[Data Service]
    F[Config Service]
  end

  subgraph "Data Layer"
    G[(User DB)]
    H[(App DB)]
    I[(Config DB)]
  end

  A --> D
  B --> E
  C --> F
  D --> G
  E --> H
  F --> I
```

### Styling with Classes
```mermaid
flowchart LR
  A[Normal]:::normal --> B[Success]:::success
  B --> C[Warning]:::warning
  C --> D[Error]:::error
  D --> E[Info]:::info

  classDef success fill:#10b981,stroke:#059669,color:#fff
  classDef warning fill:#f59e0b,stroke:#d97706,color:#fff
  classDef error fill:#ef4444,stroke:#dc2626,color:#fff
  classDef info fill:#3b82f6,stroke:#2563eb,color:#fff
  classDef normal fill:#6b7280,stroke:#4b5563,color:#fff
```

### Complex Sequence with Notes
```mermaid
sequenceDiagram
  participant A as Client
  participant B as API Gateway
  participant C as Auth Service
  participant D as Database

  Note over A,D: Authentication Flow

  A->>B: Login Request
  activate B
  B->>C: Validate Credentials
  activate C
  C->>D: Query User
  activate D
  D-->>C: User Data
  deactivate D

  alt Credentials Valid
    C-->>B: JWT Token
    deactivate C
    B-->>A: 200 OK + Token
    deactivate B
  else Invalid Credentials
    C-->>B: Auth Error
    deactivate C
    B-->>A: 401 Unauthorized
    deactivate B
  end

  Note over A: Token stored locally
```

```

```
A --> B     Solid arrow
A --- B     Solid line (no arrow)
A -.-> B    Dotted arrow
A -.- B     Dotted line
A ==> B     Thick arrow
A == B      Thick line
A --o B     Circle end
A --x B     Cross end
```

```
->>   Solid arrow
-->>  Dotted arrow
-x    Cross at end
-)    Open arrow
