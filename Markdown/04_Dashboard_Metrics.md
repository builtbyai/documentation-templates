### Design Principles

<div style="background: white; padding: 35px; border-radius: 20px; margin: 30px 0; box-shadow: 0 10px 40px rgba(0,0,0,0.08);">
  <h3 style="margin: 0 0 25px 0; font-size: 1.6em; color: #2c3e50;">Core Design Principles</h3>

  <div style="display: grid; gap: 20px;">
    <!-- Clarity -->
    <div style="display: flex; gap: 20px; padding: 20px; background: linear-gradient(135deg, #f0fdf4 0%, #dcfce7 100%); border-radius: 12px; border-left: 5px solid #10b981;">
      <div style="font-size: 2.5em; min-width: 50px;">üíé</div>
      <div>
        <h4 style="margin: 0 0 10px 0; color: #1f2937; font-size: 1.2em;">Clarity Over Complexity</h4>
        <p style="margin: 0; color: #6b7280; line-height: 1.7;">
          Present data in its simplest, most understandable form. Avoid unnecessary decorations that don't add value. Each metric should answer a specific question clearly.
        </p>
      </div>
    </div>

    <!-- Context -->
    <div style="display: flex; gap: 20px; padding: 20px; background: linear-gradient(135deg, #dbeafe 0%, #bfdbfe 100%); border-radius: 12px; border-left: 5px solid #3b82f6;">
      <div style="font-size: 2.5em; min-width: 50px;">üéØ</div>
      <div>
        <h4 style="margin: 0 0 10px 0; color: #1f2937; font-size: 1.2em;">Provide Context</h4>
        <p style="margin: 0; color: #6b7280; line-height: 1.7;">
          Numbers alone don't tell the story. Include comparisons (vs. last period, vs. goal), trends, and benchmarks to give metrics meaning and enable actionable insights.
        </p>
      </div>
    </div>

    <!-- Consistency -->
    <div style="display: flex; gap: 20px; padding: 20px; background: linear-gradient(135deg, #fef3c7 0%, #fde68a 100%); border-radius: 12px; border-left: 5px solid #f59e0b;">
      <div style="font-size: 2.5em; min-width: 50px;">üîÑ</div>
      <div>
        <h4 style="margin: 0 0 10px 0; color: #1f2937; font-size: 1.2em;">Maintain Consistency</h4>
        <p style="margin: 0; color: #6b7280; line-height: 1.7;">
          Use consistent color schemes, typography, spacing, and component styles throughout. This reduces cognitive load and makes dashboards easier to scan and understand.
        </p>
      </div>
    </div>

    <!-- Action-Oriented -->
    <div style="display: flex; gap: 20px; padding: 20px; background: linear-gradient(135deg, #fce7f3 0%, #fbcfe8 100%); border-radius: 12px; border-left: 5px solid #ec4899;">
      <div style="font-size: 2.5em; min-width: 50px;">‚ö°</div>
      <div>
        <h4 style="margin: 0 0 10px 0; color: #1f2937; font-size: 1.2em;">Enable Action</h4>
        <p style="margin: 0; color: #6b7280; line-height: 1.7;">
          Dashboards should drive decisions. Make it clear what actions are needed, highlight anomalies, and provide drill-down capabilities for deeper investigation when needed.
        </p>
      </div>
    </div>
  </div>
